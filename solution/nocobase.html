Overview

AIが作り、人が見たまま改善できる業務アプリ基盤。

NocoBaseは、データモデル、画面、権限、ワークフローなど業務システムに必要な基盤機能を持つノーコード/ローコード開発プラットフォームです。AI Coding Agentによる初期構築と、現場担当者によるノーコード改善を組み合わせられる点が特徴です。

Rakucloudでは、NocoBaseを単なるツール導入ではなく、業務設計・データ構造・運用ルールまで含めた業務基盤として設計します。

AI Coding Agent

初期構築を高速化

CodexやCursorなどのAI Coding Agentを活用し、データモデル、ページ、ワークフローのたたき台を短期間で作成します。

No-Code UI

現場で改善しやすい

設定モードで画面や項目を見ながら調整でき、運用後の変更にも対応しやすくなります。

分析、質問応答、文書処理、タスク振り分けなどを業務データと権限の範囲内で支援します。

Workflow

業務処理を自動化

Architecture

AI生成だけに依存しない、運用に耐える基盤設計。

業務システムは、作る速さだけでなく、権限、監査、データ整合性、運用変更への追従が重要です。NocoBaseの基盤機能を使い、AIと人が同じデータモデル上で改善できる構造を作ります。

01

データモデル駆動

UIとデータ構造を分離し、複数画面・複数業務から同じデータを安全に扱えるようにします。

02

プラグインアーキテクチャ

標準プラグインと個別開発を組み合わせ、業務固有要件に段階的に対応します。

03

権限・監査・ワークフロー

ロール、フィールド単位のアクセス、操作ログ、承認・通知フローを現場運用に合わせます。

04

オープンな外部連携

HTTP API、CLI、外部Agent、SaaS連携を使い、既存システムとつながる業務基盤にします。

FAQ

よくある質問

NocoBaseは自社サーバーやAWSで運用できますか？

可能です。Docker構成、クラウド環境、バックアップ、監視、権限設計まで含めて運用前提で設計します。

ノーコードだけで足りない場合も対応できますか？

対応できます。標準機能で構築できる範囲を優先し、個別要件はプラグイン開発やAPI連携で補完します。

既存のExcelやSaaSから移行できますか？

移行できます。既存データの棚卸し、データモデル設計、移行手順、運用開始後の改善まで支援します。
